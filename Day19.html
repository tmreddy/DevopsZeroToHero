Jenkins Zero to Hero

  1. Install Jenkins on ec2
    1. Jenkins
    2. Launch Ubuntu instace
    3. SSH to ec2 instance
    4. Jenkins is a java based so we need to install java sdk
    5. sudo apt update
    6. sudo apt install openjdk-11-jre
    7. java -version
    8. curl -fsSL https://pkg.jenkins.io/debian/jenkins.io-2023.key | sudo tee \ /usr/share/keyrings/jenkins-keyring.asc > /dev/null echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \ https://pkg.jenkins.io/debian binary/ | sudo tee \ /etc/apt/sources.list.d/jenkins.list > /dev/null
    9. sudo apt-get update
    10. sudo apt-get install jenkins
    11. ps -ef | grep jenkins
    12. By default your ec2 instance will not accept any traffic from external world
    13. By default, Jenkins will not be accessible to the external world due to the inbound traffic restriction by AWS. Open port 8080 in the inbound traffic rules as show below.
    14. EC2 > Instances > Click on
    15. In the bottom tabs -> Click on Security
    16. Security groups
    17. Add inbound traffic rules as shown in the image (you can just allow TCP 8080 as well, in my case, I allowed All traffic).
    18. On browser type public ip address of ec2 install with port 8080
    19. you can get password from this sudo cat /var/lib/jenkins/secrets/initialAdminPassword
    20. Click on Install suggested plugins
    21. Create first admin user
    22. Jenkins installation is successful.
    23. Jenkins master should be used only for scheduling
  2. Configure Jenkins and expose to outside world
  4. Install docker on the same EC2 instance
  5. $ sudo apt install docker.io
  6. Grant Jenkins user and Ubuntu user permission to docker deamon.
  7. sudo su -
    usermod -aG docker jenkins
    usermod -aG docker ubuntu
    systemctl restart docker
  8. $su -jenkins
  9. $ docker run hello-world
