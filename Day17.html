  10. Install Terraform
    1. Mac
      1. Brew tap hasicorp/tap
      2. brew install hashicorp/tap/terraform
      3. brew upgrade hashicorp/tap/terraform
    2. Linux
      1. sudo apt-get update && sudo apt-get install -y gnupg software-properties-common
    3. $ terraform -version
    4. $ terraform -help
    5. Terraform works in four steps
      1. Terraform init - Initialize
      2. Terraform plan - dry run
      3. Terraform apply
      4. Terraform destroy
    6. vim main.tf
    7. $ terraform init
    8. Always maintain inputs.tf for defining variables and outputs.tf files
    9. $ terraform plan
    10. Pre-request for terraform to work with AWS provider is to configure aws cli configure
    11. $ terraform apply
    12. terraform.tfstate file maintaints the state of the infrastructure it created
    13. You should store your state files remotly, not on your local machine
    14. It is not a good idea to store the state file in source control
    15. Isolate and organize the state files to reduce the blast radius
    16. Do not manipulate the state file

  15. Problems with terraform
    1. State file is a single source of truth
    2. Manual changes to the cloud provider cannot be identified and auto-corrected. Terrform is not bi-directional means if any one changes the resources on aws Terraform will not know
    3. Not a GitOps friendly tool. Don't play well with Flux or Argo CD
    4. Can become very complex and difficult to manage
    5. Trying to position as a configuration management tool as well
